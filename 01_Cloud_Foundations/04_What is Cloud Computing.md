# What is Cloud Computing?

## Learning Objectives

After completing this lesson, you should be able to:

- Define Cloud Computing

---

# Cloud Computing

## Definition

Cloud Computing is the on-demand delivery of IT resources over the internet with pay-as-you-go pricing.

Instead of buying and maintaining physical servers and infrastructure, users can access computing resources whenever needed through a cloud provider.

### Examples of Cloud Resources

- Servers
- Storage
- Databases
- Networking
- Software Applications
- Analytics Tools
- Artificial Intelligence Services

---

# Key Characteristics of Cloud Computing

## 1. On-Demand Self-Service

Users can provision resources whenever needed without human intervention.

### Example
Creating a virtual server in minutes through a web console.

---

## 2. Broad Network Access

Services are accessible over the internet from various devices.

### Devices
- Laptops
- Smartphones
- Tablets
- Desktop Computers

---

## 3. Resource Pooling

Cloud providers share resources among multiple customers.

### Benefit
Efficient use of infrastructure.

---

## 4. Rapid Elasticity

Resources can scale up or down quickly based on demand.

### Example
An online store adding more servers during a sale.

---

## 5. Measured Service

Users pay only for what they use.

### Examples
- Storage used
- Compute hours consumed
- Network bandwidth utilized

---

# Benefits of Cloud Computing

## Cost Savings

- No large upfront hardware investment
- Pay-as-you-go pricing

## Scalability

- Increase or decrease resources easily

## Flexibility

- Access resources anytime, anywhere

## Reliability

- High availability and redundancy

## Speed

- Deploy resources within minutes

## Global Reach

- Serve customers worldwide

# Cloud Service Models

Cloud services are generally divided into three main models:

## 1. Infrastructure as a Service (IaaS)

### Description

Provides basic computing infrastructure over the internet.

### Customer Manages

- Applications
- Data
- Runtime
- Operating System

### Provider Manages

- Servers
- Storage
- Networking
- Virtualization

### Examples

- Amazon EC2
- Microsoft Azure Virtual Machines
- Google Compute Engine

### Use Case

Organizations needing full control over infrastructure.

---

## 2. Platform as a Service (PaaS)

### Description

Provides a platform for building and deploying applications.

### Customer Manages

- Applications
- Data

### Provider Manages

- Infrastructure
- Operating System
- Runtime Environment
- Middleware

### Examples

- AWS Elastic Beanstalk
- Google App Engine
- Microsoft Azure App Service

### Use Case

Developers focusing on coding without managing servers.

---

## 3. Software as a Service (SaaS)

### Description

Provides fully managed software applications over the internet.

### Customer Manages

- Application Usage
- Configuration Settings

### Provider Manages

- Entire application stack

### Examples

- Gmail
- Microsoft 365
- Salesforce
- Dropbox

### Use Case

Users wanting ready-to-use software.

---

# Cloud Deployment Models

Deployment models define where cloud resources are located and who owns them.

---

## 1. Public Cloud

### Description

Cloud infrastructure owned and operated by a third-party provider.

### Characteristics

- Shared infrastructure
- Internet-based access
- Cost-effective
- Highly scalable

### Examples

- AWS
- Microsoft Azure
- Google Cloud

### Benefits

- Lower costs
- Easy scalability
- Minimal maintenance

---

## 2. Private Cloud

### Description

Cloud infrastructure dedicated to a single organization.

### Characteristics

- Exclusive use
- Greater control
- Enhanced security

### Benefits

- Better compliance
- Increased customization
- Higher security

### Drawback

- More expensive

---

## 3. Hybrid Cloud

### Description

Combination of Public and Private Clouds.

### Characteristics

- Data and applications move between environments
- Flexible deployment

### Benefits

- Balance between cost and security
- Better workload management

### Example

Sensitive data in private cloud and public website in public cloud.

---

# Deployment Model Comparison

| Model | Ownership | Access | Cost | Security |
|---------|----------|--------|------|----------|
| Public Cloud | Cloud Provider | Public Internet | Low | Moderate |
| Private Cloud | Single Organization | Restricted | High | High |
| Hybrid Cloud | Mixed | Mixed | Medium | High |
